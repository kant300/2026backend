 메소드(함수)
 객체지향프로그램에서는 메소드
 구조적언어(C, pascal, visualBasic), 응용프로그램(엑셀)에서는 함수
 
 하나의 기능을 구현한 작은 프로그램
 초기작성시간이 오래걸린다.(작업별 분리->메소드간에 데이터 전달 통일)
 유지보수시간이 짧아진다.

1. 기본 작성
   접근제어자 (static) 반환타입 메소드이름(매개변수) {
      명령문;
      return 반환값;
   }

   ※ static는 생성을 하지 않고 바로 사용할 때 
      클래스를 상속을 통해서 사용할 때
      클래스를 생성해서 사용할 때
      외부에서 사용안하고 내부에서만 사용할 때
2. 접근제어자
                   다른 패키지   같은 패키지  클래스
      public            O           O       O    어디서나 사용이 가능
      protected         X           O       O    같은 패키지(폴더)내에서 사용이 가능   
      default           X           O       O    접근제어자를 생략했을 때
      private           X           X       O    같은 클래스내에서만

3. 반환타입은 데이터타입
    int, float, double, String, boolean 등 자바에서 사용하는 모든 데이터의 타입사용이 가능
    void는 반환할 데이터타입이 없는 경우
4. return은 반환타입에 맞는 값, 메소드를 호출한 곳으로 결과값을 전달
   반환타입    return   반환값(결과값)
     int    a    =    5 + 3
   데이터타입이 void인 경우에는 return은 생략한다.

5. 메소드의 주요기능 : 오버로딩
   오버로딩 : 같은메소드명으로 여러개를 구성할 수 있는 기능
   반환타입 변경불가, 메소드명은 동일하게, 매개변수(데이터타입을 다르게, 갯수를 다르게)

   public static int add(int a) {
   }
   public static int add(float a) { //데이터타입을 변경했을 경우
   }
   public static int add(int a, int b) { //매개변수의 갯수를 변경했을 경우
   }
