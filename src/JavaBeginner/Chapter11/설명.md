# 객체 지향 프로그램
## 1. 클래스

* 클래스란 : 하나의 프로그램 단위(파일구성)
            클래스내에는 변수와 메소드로만 구성
* 클래스를 사용 : 클래스를 상속하거나 생성해서만 사용가능
    1. extends : 일반 클래스 상속(1개의 클래스만 추가 가능)
    2. implements : 추상 클래스 상속(여러개의 클래스를 추가 가능)
    3. new 클래스명() : 일반 클래스 생성(여러번 클래스 생성이 가능, 일반클래스만 사용이 간으)

* 클래스이 종류
    1. 일반클래스 : 변수와 메소드로 구성된 클래스
    2. 추상클래스 : 변수와 내용이 없는 빈 메소드로 구성된 클래스(인터페이스) 

구조 : 접근제어자 class 클래스명 extends 상속클래스 implements 상속인터페이스{
      }  
    1. 접근제어자 : 클래스를 사용할 수 있는 범위(public, protected, default, private)
    2. 클래스명 : 첫글자 대문자로 지정(선언)
    3. extends : 다른 클래스를 현재 클래스에 추가해서 사용
    4. implements : 다른 클래스의 메소드 뼈대를 가져와서 현재 클래스에서 작성해서 사용

## 2. 생성자

클래스를 사용하기 위해서 클래스를 변수에 복사
Sample t = new Sample();

new 클래스명과 동일한 메소드명() -----생성자

## 3. 인스턴스 변수와 클래스 변수

인스턴스(필드) 변수 : 클래스에 선언된 변수
클래스 변수 : 생성한 클래스에 인스턴스 변수

class Sample{ ---사용전 클래스
    int age;    ---필드(인스턴스) 변수
    String Name;

    public Sample() { ---생성자(클래스이름과 동일한 이름의 메소드)
    }
}

class test extends Sample{
    age = 30;   ---상속받은 클래스변수(필드,인스턴스)
    name = "홍길동";
}
class test2 {
    Sample sample = new Sample(); ---클래스를 생성
    sample.age = 27;   ---클래스변수(생성한 클래스를 통해서 필드를 지정할때)
    sample.name = "홍길동";
}

## 4. 주요 추가 용어
오버로딩 : 동일한 이름의 메소드를 열거개 만들어서 사용
오버라이딩 : 추상클래스에 빈메소드를 채워서 사용
