JAVA프로그램 개발 준비

1.자바개발툴(JDK)
Oracle JDK ; 정식배포버전
Open JDK :무료(유저들이 만들어서 배포)

현재 버전은 24버전 : 안전한 버전(많은 사람이 사용했을때 큰 문제가 없는 버전) 21~22버전

웹개발해서 배포(서버에 설치해서 운영)가 목표
Tomcat 서버를 이용해서 자바를 운영
    Tomcat 9 : JDK 8버전이상
    Tomcat 10 : JDK 11버전이상

2. JDK를 운영체제가 인식할 수 있게 설정(환경설정)
JAVA_HOME에 설치된 경로 : 운영체제에서 JDK를 인식시켜주기 위해서
path에서 %JAVA_HOME%\bin : 운영체제 어디에서든 JAVA를 구동(실행/사용)시키기 위해

윈도우 제어판-정보-고급시스템 정보-환경변수

3. 편집기(프로그램 작성하는 도구)
Visual Code : 통합편집기(여러 언어 프로그램이 가능한 도구)-무료-프로트엔드 개발자가 주로 사용
                HTML, javaScript, Java, C, Pthyon
Eclipse : 자바 전용 편집기-무료-개발중단, 실무에서 정부프레임워크 개발에 주로 사용
Intellij : 자바 전문 편집기-유료-년단위 4번 업데이트, AI접목, 개발에 필요한 프로그램 연동
            데이터베이스, github(깃허브), Docker(도커, 가상환경), Tomcat서버 내장
메모장, Edit Plus

4. 프로그램
.java --------------- .class -------------.jar
사용자작성프로그램     컴파일된 파일           배포파일  
.html : 웹페이지 문서
.js   : javaScript
.css  : 스타일시트(웹문서의 모양, 색상등을 지정)

asset : 프로그램 개발에 필요한 조각 프로그램, 자원(이미지, 아이콘, 음악) 관리

5. 자바 프로그램 작성순서
프로젝트(하나의 개발프로그램)>패키지(폴더, 그룹으로 파일관리)>클래스(파일, 하나의 프로그램 단위)
>메소드(하나의 기능)>변수, 명령어

6. 실행(인텔리제이에서 작성한 프로그램을 구동)
   디버깅(작성한 프로그램을 추적해서 오류를 수정하는 것)
   빌드(작성한 프로그램을 최종 배포파일(jar)로 만드는 것) 
7. 자바프로그램 문법
   접근자 class 클래스명 { --- 클래스 생성, 클래스 정의, 클래스 만들기, 클래스 선언
        접근자 전달자 메소드명(매개변수){ --- 매소드생성, 정의, 만들기, 선언
            변수선언; --- 문장의 끝에는 ; 표시
            입력처리{ --- 범위는 { } 묶어서 사용
            }; --- } 뒤에는 ; 을 생략
            작업처리;
            출력처리;
        }
   }

8. 프로그램 해석 - 오류해결을 못한다.
기획 : 클래스-메소드 이름으로 작업을 분리
작성 : 메소드-명령어를 해석해서 동작을 해석
