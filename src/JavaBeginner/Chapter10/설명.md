예외처리
일반적인 프로그램은 프로그램의 오류가 발생하면 오류수정후 프로그램 정상적으로 사용이 가능.
오류가 해결이 안되면 프로그램 사용이 불가능 ==> 문법, 규칙

사용 중 부정확한 데이터로 인해 발생하는 오류(이용상의 문제발생)가 발생했을 때
해당 오류를 무시하고 정상적으로 프로그램 사용할 수 있게 하는 기능

1. 문법
   try {
     정상적으로 처리할 프로그램(작성한 프로그램)
   } catch(오류종류) { ---해당오류가 발생하면 예외처리(오류를 무시)
     정상적으로 작업처리 중 해당 오류가 발생했을 때 처리할 프로그램(오류메세지)
     오류메세지를 모아서 관리 : log(로그)
   } finally {
      정상처리 또는 오류처리 후 처리할 프로그램(일반적으로 생략)
   }

   2. 오류종류
   Exception                        모든 오류가 발생했을 때 예외(*)
   RuntimeException                 실행 중 오류가 발생했을 때 예외
   NullPointerException             null(값이 없음) 객체 사용했을 때 예외(*)
   ArrayIndexOutOfBoundsException   배열 범위를 초과했을 때 예외
   ArithmeticException              0으로 나누었을 때 예외(산술연산시 오류)(*)
   NumberFormatException            문자열을 숫자로변환시 오류가 발생했을때 예외
   IOException                      입출력 오류가 발생했을 때 예외(파일입출력)(*)
   FileNotFoundException            파일이 존재하지 않을 때 예외
   SQLException                     데이터베이스 처리시 오류가 발생했을 때 예외
